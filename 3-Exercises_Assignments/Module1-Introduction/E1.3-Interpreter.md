Open a terminal in the folder that contains `hello.py` and run (use `python3` on modern macOS):

```bash
python3 hello.py
```

Expected output:

```

```

Behind the scenes:

1. The interpreter parses your source code.
2. Python compiles the parsed code to bytecode (this is the `.pyc` cached form).
3. The Python Virtual Machine (PVM) executes the bytecode and produces the program output.

---

#### Step 3 â€” Generate and inspect the compiled bytecode ğŸ”§

To generate a `.pyc` file explicitly, use:

```bash
python3 -m py_compile hello.py
```

This writes the compiled bytecode into `__pycache__/`.

List the cache to confirm a `.pyc` was created:

```bash
ls -l __pycache__
# Example output:
# -rw-r--r--  1 user  staff  1234 Oct 11 12:00 hello.cpython-3xx.pyc
```

View the bytecode instructions

To see the low-level bytecode instructions Python generates, use the `dis` module:

```bash
python3 -m dis hello.py
```

Quick note: `dis` prints the opcode-level instructions (e.g., LOAD_CONST, LOAD_NAME, CALL_FUNCTION). The output is intentionally low-level and useful when you want to demonstrate how Python executes code.
