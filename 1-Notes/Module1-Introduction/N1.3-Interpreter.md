## ğŸ§  Python Interpreter Architecture

It includes multiple components that work together to execute Python code:

- **Parser:**  
  Reads the code line by line and checks if your code is written correctly following Python's syntax rules (detects syntax errors).

- **Compiler:**  
  Converts the parsed code into bytecode (from `.py` to `.pyc`), which is a lower-level, platform-independent representation, also called as machine executable code.

- **Python Virtual Machine (PVM):**  
  Takes the bytecode and executes it. This is where the actual logic runs, line by line.  
  And finally you get the output of the program.

---

<a id="what-is-an-interpreter"></a>
## So, What is an Interpreter?

- The interpreter is like the whole machine running your Python code from start to finish.
- It's the overall system that reads the code, compiles the code, and executes the code.

---

<a id="how-python-interpreter-works"></a>
## What happens when you run a Python program? OR How Python Interpreter Works?

1. The **Python interpreter reads your code line by line** and runs it directly.  
2. Behind the scenes, Python **automatically compiles your code into bytecode** (a simpler internal step).  
3. This bytecode is then run by the **Python Virtual Machine (PVM)**.

- **Python is mainly an interpreted language.**
- Because compilation is **automatic** and happens in the background (unlike Java, you don't compile the code manually), Python is considered an **interpreted language** from a user's point of view.

## âœ… Note

- While Python is called an interpreted language, it actually performs **compilation internally**.
- The Python interpreter first **compiles the code to bytecode**, but this step is hidden from the user.
- In contrast, languages like Java require the programmer to **explicitly compile** the code before execution.
- ğŸ” Therefore, since Python doesn't require a separate compilation step, it is commonly referred to as an **interpreted language**.
- However, in loose terms, Python is often described as **both compiled and interpreted**, because the source code is **compiled to bytecode**, which is then **interpreted by the Python Virtual Machine (PVM)** at runtime.
